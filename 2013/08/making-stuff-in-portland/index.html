Hobby electronics in Portland

August 2013 - Portland, Oregon

A technician at work recently asked me about getting started making stuff in Portland and in general so I braindumped into an email. I’ve cleaned up and added a lot before posting it here. These are only things I’ve actively been to or seen, which means there’s a whole city of awesome out there.

Community

First off, Portland is pretty awesome and it’s got an awesome calendar. Keep an eye on Calagator, the Portland Tech calendar.

You might want to get a GitHub account. Feel free to follow ‘wicker’.

If you have a Monday evening free, all kinds of electronics hackers bring their projects to Dorkbot on every other Monday at Backspace Café in downtown Portland.

The Electronics Prototyping Lab at Portland State has walk-in lab hours to use their laser cutter, 3D printer, LPKF board router, soldering stations, and reflow oven.

There’s the PDXDrones group with a monthly fly-out and invited speakers as well as the Portland State Aerospace Society building rockets and related avionics every Tuesday night on campus.

Linux

I’ve learned a ton about how computers, electronics, and networking works by being able to look under the hood of my laptop.

If you’re not comfortable with dual-booting your current laptop, where you add Linux alongside your current operating system and choose one every time you boot up, then I suggest buying a second-hand laptop running Ubuntu Linux from FreeGeek’s thrift shop on SE Hawthorne. They run a couple hundred dollars and you’re supporting a great cause, but if that’s too steep you can also volunteer in their Build Program to earn your own.

Using Ubuntu, Debian, Fedora, Mint, or whatever brand of Linux you prefer, get set up with SSH keys and be able to clone git repositories from GitHub. Read all the GitHub help documentation.

Looking around GitHub, you’ll find the same two files over and over. One is README and by looking you’ll see what should be in a good README and what’s lacking in the bad ones. The other is a LICENSE file and this is where you’ll tell other developers what you do and don’t want them to do with your code.

Open Source Licenses

I feel strongly that one of the great things about the electronics maker community is in being very pro-open source where source code and hardware schematics are made freely available. I don’t just want to know what you made, I want to know how you made it.

I do want to note that there are a vocal crowd out there that says all software should be free. Do them the favor of reading up on their points of view and adjust your philosophy accordingly. Then realize that the way things are right now is really open source instead, where you choose a license that allows others to use your code and schematics but limit them to obey certain conditions.

You can read up on the current open source licenses and pick one that you like. LadyAda has an intro to open hardware licenses, which is something that’s still evolving.

I tend to license my hardware and software under the 3-clause BSD license, which says you can take my work but you must include the copyright notice and my name whether you’re releasing (a) my source code or (b) a compiled binary, and © you can’t use my name or the name of my orgnaization to endorse or promote products you make by changing my software.

Make Your Own Boards

It’s typical to start out with a breadboard but pretty soon you’re going to want something a little more durable. Also, you’ll want to put the project up on a shelf and re-use that breadboard.

Make your own boards by designing your circuit and laying out the traces on your own circuit board in a schematic/board layout editor using Eagle or KiCad. You’ll probably want to start with 2-layer boards so don’t pay money for the standard version of Eagle until you’re absolutely sure you want it. The hobby version lets you build small 2-layer boards for free.

I recommend not starting out on Eagle but, if you feel you must, don’t think you can’t switch later. Any good open source schematics editor will open Eagle files. I started out with Eagle, got the standard Eagle, and while I like the program, I got somebody else to pay for it and would not have paid for it myself. Plus it’s closed source, I was never able to get a copy of the license, and their customer service is nonexistent.

Get your board fabbed locally for $5/sq inch through OSHPark by ordering online, uploading your CAD files, and getting it mailed to your house in a couple of weeks.

When you’re designing the board, figure out your bill of materials using Digikey, Mouser, Newark, Sparkfun, and Adafruit.

Always order the parts when you order the board to be sure everything’s in stock. Double check your packages and footprints so the parts you order will fit your board!

Digikey tip: order it USPS regular shipping, you generally get it in like two days for $5. Consider overnight shipping a last resort.

Are you completely lost and confused when it comes to laying or building boards? The Electronics Prototyping Lab at Portland State has a calendar of open lab walk-in hours.

For local parts retailers, make the pilgrimage to Oregon Electronics in Beaverton, Norvac in Salem, or Fry’s in Wilsonville.

All the places above link you to the datasheet. Read every datasheet. Start to learn the words.

Use a Development Platform

For basic introduction to C on microcontrollers, you really can’t beat a basic Arduino.

For FPGA, go with a Papilio kit or an Altera DE0-Nano from Adafruit and check out the FPGA4Fun for a place to start.

There are a great set of introductory tutorial systems at Learn at SparkFun and Adafruit Learning.

Once you’ve maxed out your Arduino, move on to a Beagleboard or Raspberry Pi.

Two common, beginner-friendly embedded operating systems are the Robot Operating System and ChibiOS.

For fun project ideas and community, check out Make Magazine and the Hackaday Blog.

Writing Code

Need to learn a new language or framework? Pick a language, find a challenge, start writing code, and commit commit commit. Your early code can be terrible. One of the best ways to get good is to write a lot of code, write mountains of code, and keep it so you can look back and learn from it.

For example, I chose C and wrote a bunch of simple solutions for Project Euler.

Jennifer DeWalt is learning website front-end by building 180 Websites in 180 Days.

Knowledge

Once you’ve done some random projects, get more systematic.

Try to do a project for each protocol, from I2C to SPI to serial.

Read all the Wikipedia pages and be able to draw the different protocol timing diagram shapes.

Learn when and when not to use each. Start to learn what physically works and what doesn’t.

Ask not only ‘what is this unfamiliar term?’ but also ‘what are its alternatives?’.

The Navy Electricity and Electronics Training Series is available for free.

One place for explanations/answers is the Electronics StackExchange.

The big online resource for drones is DIYDrones.
