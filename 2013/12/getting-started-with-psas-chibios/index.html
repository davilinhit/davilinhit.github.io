Using ChibiOS and OpenOCD

December 2013 - Portland, Oregon

I chose ChibiOS for the GPS RF board because PSAS has it set up. These are rough notes for myself for doing development on the GPS RF board.

The stm32 section on the PSAS wiki was useful.

/opt/psas is where I’ve put all the psas ChibiOS stuff on my machine.

I put the following commands in ~/.bash_aliases:

alias psas_stm="echo Path is $PATH && export PATH=$PATH:/opt/psas/openocd/bin:/opt/psas/x-tools/stm32f407/arm-psas-eabi/bin:/opt/eclipse/ && echo \nNew path is \$PATH"

alias gpsdev1="cd /opt/psas/openocd/bin && ./openocd -f ../share/openocd/scripts/interface/stlink-v2.cfg -f ../share/openocd/scripts/target/stm32f4x_stlink.cfg"

alias gpsdev2="cd /opt/psas/openocd/bin && telnet localhost 4444"

To program the board, first I make sure I’ve updated the software in the main repo, which in GPS case is here.

Not sure where my code is actually compiled but I’m going to use the ch.hex that’s created when I do compile to program the board.

Then I run psas_stm, which adds the paths I need.

Then I open two terminals.

In the first, I run gpsdev1 to run openocd on the config files.

In the second, I run gpsdev2 to open communication with the board.

The openocd commands to run inside the second terminal are

reset halt
flash write write_image erase /home/wicker/proj/psas/stm32/src/gps/build/ch.hex
reset

Then the board should be programmed correctly.

Huge thanks to Kay and Dave for helping me get started.
