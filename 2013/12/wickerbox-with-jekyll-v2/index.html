Making Wickerbox - v2

Wickerbox is a dynamically-generated static site served using Jekyll and Github Pages. I recently redesigned it with an eye to a more simple back-end and a minimum of Javascript. I ended up directly linking from Google+ Photos for photo posts and from Github for project posts.

Wickerbox is static in that it does not require a server-side database in order to serve up pages. I came from Wordpress using MySQL and PHP and wanted something simpler. (ha!) Also, Iâ€™m a big fan of git for revision control.

What’s a Post?

A post is a post is a post. I originally split posts into different layouts with photo posts containing a gallery sourced from a Picasa photo album so I could just put the userid and albumid in the header and it would automatically populate. That ended up being really awkward to browse and I prefer just putting the photos in the document. It works for projects, too.

I write the posts in Textile and save as .textile documents, which are then turned by Jekyll into HTML pages. Jekyll takes any otherwise-unmarked piece of text and renders it with paragraph tags, which I gave some padding to get the indented effect.

By starting the line with two spaces, I can avoid wrapping my images in paragraph tags and make sure they take up the entire width of the content box.

Posts do need to be sorted for display purposes in the category and front pages, so I’m still using a lot of header:

Implementation

I wrote the layout file to be as simple as possible. I’m stopped using an out-of-the-box responsive framework and intend to just put in some mobile tweaks some time in the future. I’m not too worried about making the site look fantastic on mobile because the photos are integral and they’re huge, so I’m not really sure who’s spending a lot of time on mobile browsing.

I stopped using the Galleria plugin for photos but I did switch fonts to use Google.

My intent was to make posts as simple and standard as possible while putting the complicated sorting and display details on the front page and on the category pages. The goal was to lower the transaction cost of writing a post as much as possible.

To Tag or Not To Tag

I’m still adding tags to posts but I’m not displaying them because there isn’t a nice design-friendly place to put them. I used to love tag clouds; now I’m sort of ambivalent.

Notebook

The biggest thing I wanted was to make a blog-like page which would display all the posts under the ‘notes’ category. It would be fine to drop photos here but the goal was to lower the transaction cost even lower so I’d err on the side of a quick post instead of letting all these ideas pile up until I burn the whole pile and vow to start over, which is basically what happened in 2013.

RSS Feed

I’m using the basic template from snaptortoise’s Jekyll RSS Feeds repo and it works fine.

Source
