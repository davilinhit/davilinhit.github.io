Friday, January 3 2014

Caution: contents contain notes about the Eagle CAD program. Readers allergic to Eagle should skip it.

I use the standard version of Eagle for making printed circuit boards. It’s basically a cad program you can use to draw the schematic and then route the layout of the final board. I usually take my files and send them off to OSHPark to be made into real physical boards.

I don’t recommend Eagle anymore; I’ve heard too many good things about KiCad, which is open source and free. Go learn that instead if you haven’t already put time and money into Eagle. I’ve had customer support problems and nobody ever actually gave me a copy of the Eagle license that they made me pay money for. They also misrepresented what an academic license consisted of. So go to KiCad.

This note’s about the eagle.scr startup script which sets default values for things like via diameter, wire size, and grid size. I look to OSHPark’s design rules to find out what these values ought to be. The fab houses that receive my design files and fabricate my boards for me set the minimum widths, distances, diameters, etc that their machinery can accommodate considering how much money I want to give them. I like to set my minimum settings a little larger so I don’t run the risk of small inaccuracies breaking my design but I can go smaller if I need to. (I don’t usually need to.)

I’ve been experimenting a little with the Eagle startup script. I was using semi-colons inconsistently and it turns out if you have multiple statements on one line, you need to separate them with a semi-colon. Either a semi-colon or a newline appear to work for statements in list form. For example, this works fine:

change drill 0.013; change diameter 0.027
change shape round;

This doesn’t:

change drill 0.013 change diameter 0.027

But this does:

change drill 0.013
change diameter 0.027

The script allows you to set default values for the board, schematic, device, symbol, package, and general library editing screens. The package screen is basically the same as board except there’s no need to specify the route width because there isn’t any routing.

I work in inches because that’s how I started out and the important thing is just to be consistent with grid sizes and measurement system.

My script looks like this:

OVERALL
GRID IN 0.0125 DOTS ON ALT IN 0.025;

BOARD
change drill 0.013 # drill/hold diameter
change diameter 0.027 # via pad diameter
change shape round # via shape
change width 0.01 # route width
change size 0.50 # text size

SCH
change width 0.006 # wire width
change size 0.50 # text size

No changes needed for library or device

SYM
change length middle # default pin length
change visible both # pin and pad number

PAC
change shape round # via shape
change diameter 0.027 # via pad diameter
change drill 0.013 # drill/hold diameter
change size 0.050
